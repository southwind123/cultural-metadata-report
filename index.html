1. 完整性 (Completeness)

1.1 元数据完整性（A1）

定义：
检查“字段层面”是否具备一普参考的全部核心字段，而不是逐条记录。
用户提供的一普必填项为：
藏品编号、藏品名称、藏品类型、文物级别、文物类别、质地类别、文物来源、年代、入藏时间范围、完残程度、质量范围。
本指标的不合规率按“缺失字段数 / 规则要求字段总数”计算。

1.1.1 检查内容

  • 对照一普必填项，逐项核对本次数据集中是否存在对应字段（仅按字段层面，不逐条看记录是否为空）。
  • 将本次数据的 7 个字段与一普字段做映射：
    • 藏品编号 → 一普“藏品编号”
    • 藏品名称 → 一普“藏品名称”
    • 年代 → 一普“年代”
    • 质地 → 一普“质地类别”（名称略有差异，但语义接近）
    • 类别 → 一普“文物类别”（字段名不完全一致）
    • 级别 → 一普“文物级别”
    • 尺寸 → 一普中无完全对应的必填项，可视为本馆扩展字段

1.1.2 质量问题

（1）【问题A1-1】一普核心字段在本数据集中缺失（字段层面）

  • 说明：
    一普要求的 11 个核心字段中，本数据集中完全不存在以下 5 项，对后续统计与共享将形成结构性缺口。此为“全局性问题”，影响所有记录。

  • 缺失字段列表（字段名层面）：

    1. 藏品类型：完全没有对应字段，无法区分藏品按来源、用途等的大类。
    2. 文物来源：无字段记录征集方式、来源渠道，影响 provenance 管理和合法性审查。
    3. 入藏时间范围：无字段记录何时入藏，影响馆史研究和资产管理。
    4. 完残程度：无字段记录文物是否残缺、残缺比例，影响修复、展陈决策。
    5. 质量范围：无字段记录文物品质等级（如是否精品），影响重点保护与展览选品。

（2）【问题A1-2】部分字段命名与一普不完全对齐，语义易混淆（字段层面疑似问题）

  • 说明：
    以下字段虽可大致映射到一普字段，但命名不完全规范，容易在跨系统共享时产生歧义。也为“全局性问题”。

  • 具体情况：

    • 字段 “类别”：
      • 一普中常见字段名为“文物类别”或“门类”。
      • 目前仅用“类别”，且内部既有“玉器、铜器”等材质门类，又有“绘画、书法”等艺术门类，还有“生活用品类、杂类”等用途/残余类目，语义混合，不够清晰。
    • 字段 “质地”：
      • 一普中多用“质地类别”或“材质”，目前仅用“质地”，但语义基本可理解。
      • 建议在元数据标准中明确与“质地类别”的对应关系。

本小节为字段层面问题，故不按记录逐条列举；视为对全部 100 条记录均产生影响。

1.1.3 不合规率（A1）

  • 一普要求核心字段总数：11 个
  • 本数据集中完全缺失的核心字段数：5 个
  • 不合规率（字段层面） = 5 / 11 ≈ 45.45%

1.2 内容完整性（A2）

（1）指标定义

  • 在现有 7 个字段中，检查每条记录是否“字段值为空/缺失”：
    • 理解为："" 或明显缺省视为缺失；
    • 不对“值内容是否合理”作判断（那是 C1 / D1）。
  • 若某条记录至少有一个核心字段值缺失，则这条记录在 A2 下“不完整”。

（2）实际检查内容

  • 针对每条记录的 7 个字段：
    • 核查是否为空字符串或缺字段；
    • 本次数据中 7 个字段在结构上都存在，因此只检查字段值是否为空。

2. 规范性 (Standardization)

2.1 数据格式规范性（B1）

定义

  • 检查各字段的数据格式、书写口径是否统一，包括：
    • 编号规则（长度、结构）
    • 年代字段的书写模式
    • 质地字段的多材质表示方式
    • 类别字段用字是否统一
    • 尺寸字段的符号、结构
  • 不再重复统计“空值”（空值在 A2 已统计），重点关注“有值但格式不统一/不规范”的情况。

本次实际检查内容

  • 字段范围：
    • 藏品编号：长度与结构是否一致。
    • 年代：是否统一采用“纪年+起讫年”格式。
    • 质地：多材质记录中分隔符是否统一。
    • 类别：用字“其他/其它”等是否统一。
    • 尺寸：乘号符号（*、＊、x）、多件尺寸表示方式等。
  • 空值不再在本指标下列为问题，以避免与 A2 重复。

9. 通俗总结与改进建议

从馆内日常工作角度看，这批数据的主要问题集中在三块：

  1. 字段层面“少了重要栏位”

    • 一普标准要求的“文物来源、入藏时间范围、完残程度、质量范围”等字段在当前表中 压根没建出来。
    • 意味着：
      • 无法用这张表做“来源结构分析”（自征/征集/调拨/考古出土等）；
      • 无法快速筛选“破损严重/需修复”的文物；
      • 无法对入藏时间进行统计（例如近十年新入藏文物）。
    • 建议优先级：最高（系统建模级别）
      • 在系统中按一普标准补建对应字段；
      • 制定录入规范，为已有藏品逐步补录这些核心信息。

  2. 分类和格式“各写各的”，难以支持统计与共享

    • 类别口径杂乱：
      • 鼻烟壶被分到“生活用品类、玉器、杂类、瓷器”等不同类目；
      • 印章有的算“石器/玉器”，有的叫“印信、符牌”；
      • 大量使用“杂类、其它”，分类含义模糊。
    • 格式不统一：
      • 年代有“清(1616~1911)”“新石器时代”“其他”“中华人民共和国(1949年10月1日成立)”等多种写法；
      • 尺寸字段混用“*、＊、x、空格”，多件套写法各不相同，还掺杂换行；
      • 编号流水位数也有不统一的情况。
    • 这些问题会导致：
      • 做统计时，同一类文物被分散到多个类别；
      • 跨系统对接（导入导出、程序解析）容易出错或需要大量清洗。
    • 建议优先级：高
      • 确定若干关键字段的“馆内统一规范表”（如：年代、类别、质地、尺寸格式）；
      • 对鼻烟壶、印章等重点类别，先统一一套分类口径，再分批校正；
      • 对尺寸字段，考虑在新系统中拆为结构化字段（长/宽/高/单位/件数）并逐步清洗旧数据。

  3. 少量记录存在“看上去就不对”的数值和用词

    • 如银质章直径“340”（疑似漏小数点）、“尺寸规格不一”“大小不一”之类笼统描述；
    • 个别记录出现“裤高”“□”“囗囗”等未解释的字样和占位符。
    • 这类问题虽然数量不多，但一旦用于出版、展示或科研，容易被同行质疑数据严谨性。
    • 建议优先级：中
      • 针对不合理尺寸（如圆径340）逐一核对原始量测记录；
      • 对“尺寸规格不一/大小不一”类记录，至少补充典型尺寸或范围；
      • 对“□”“囗囗”做好说明：在说明字段中解释是残缺还是难读铭文，并统一标注格式。

整体改进路径建议：

  1. 先补“缺的字段”，再调“写法”和“分类”

    • 先在系统中补齐一普要求的字段（来源、入藏时间、完残等），为后续工作留“位置”。
    • 同时确定一套简明可操作的字段规范文档。

  2. 优先治理“高频跨系统使用”的字段

    • 如：年代、类别、质地、尺寸、级别、藏品编号。
    • 从本次问题清单中，先挑出同类文物（鼻烟壶、印章、辽代铜鎏金器）做一批“示范整治”，形成可复制经验。

  3. 建立“新录入即规范”的机制

    • 对编目员提供下拉选项和格式校验（如自动补全年代格式、统一乘号符号），避免产生新的不规范数据。
    • 对编号规则、类别词汇等设定系统级校验，减少人工随意性。

  4. 结合业务需求有选择地精细化

    • 对于将来重点研究、展陈的类别（如鼻烟壶、印章、契约文书、金属器），建议优先进行精细化清洗；
    • 对短期内不常用的记录，可采用“遇用则修”的策略，逐步完善。
